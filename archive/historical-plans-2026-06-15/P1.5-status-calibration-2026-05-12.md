
---

### 2.2 HardwarePullerEmu (✅ 已实现)

**文件**: `plugins/gpu_driver/sim/hardware_puller_emu.cpp` (204行)

**状态机（7状态）**:

| State | Transition | Action |
|-------|-----------|--------|
| IDLE | doorbell poll → FETCH | 等待 doorbell |
| FETCH | mem_read → DECODE | 从 GPU 内存 DMA 读取 GPFIFO entry |
| DECODE | release=1 → SEMAPHORE else → SCHEDULE | 检查 release bit |
| SCHEDULE | → DISPATCH | passthrough |
| DISPATCH | scheduler->enqueue → COMPLETE | 选择 engine + 入队 |
| SEMAPHORE | release=1 → releaseSemaphore() → COMPLETE | 信号量处理 |
| COMPLETE | handleComplete → IDLE/FETCH | 提升中断，advance index |

**runLoop() 流程**:
```
while (running_):
    wait on cv (doorbell poll 或 state != IDLE)
    switch state:
        IDLE: poll doorbell → transitionTo(FETCH)
        FETCH: mem_read gpfifo entry → transitionTo(DECODE) 或 IDLE
        DECODE: if release → SEMAPHORE else SCHEDULE
        SCHEDULE: → DISPATCH
        DISPATCH: scheduler->selectEngine + enqueue → COMPLETE
        SEMAPHORE: if release → releaseSemaphore() → COMPLETE
                   else → waitSemaphore() → DISPATCH
        COMPLETE: handleComplete (interrupt raise), advance index → IDLE or FETCH
```

## 3. 集成缺口分析

### 3.1 缺口列表

| 缺口 | 位置 | 影响 | 优先级 |
|------|------|------|--------|
| `scheduler_ = nullptr` | plugin.cpp:38 | HardwarePuller 无可用 scheduler | P0 |
| GlobalScheduler 从未创建 | plugin.cpp:19 | GlobalScheduler 未实例化 | P0 |
| `setLaunchCallback` 从未调用 | - | callback 无法连接到 CudaScheduler | P0 |
| 多队列硬编码 queue_id=0 | doorbell_emu | 只支持单队列 | P1 |
| TaskRunner 符号链接可能损坏 | external/TaskRunner/ | CMake 检查通过但需验证 | P1 |

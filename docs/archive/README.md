# 归档文档说明

这个目录包含历史文档和参考资料，这些文档不再用于日常开发参考，但保留了项目的历史信息。

## 归档策略

我们将以下类型的文档放入归档：

- **规划文档**: 项目路线图、开发计划等
- **历史文档**: 已过时的设计文档
- **参考资料**: 特定主题的详细参考

## 目录结构

```
archive/
├── planning/          # 规划文档
│   ├── ROADMAP.md
│   ├── development_implementation_plan.md
│   ├── linux_driver_compatibility_plan.md
│   └── linux_driver_compatibility_test_plan.md
└── misc/              # 其他归档文档
    └── SUMMARY_CN.md
```

## 如何使用归档

1. **查找历史规划**: 查看 `planning/` 目录
2. **查找旧版本文档**: 查看对应子目录
3. **引用归档文档**: 在引用时注明"已归档"状态

## 文档状态说明

| 状态 | 说明 |
|------|------|
| **活跃** | 日常开发参考，保持在 `docs/` 根目录或分类目录 |
| **归档** | 历史参考价值，移入 `archive/` |
| **废弃** | 完全过时，不再维护 |

## 恢复归档文档

如果需要将归档文档恢复为活跃文档：

1. 从 `archive/` 移动到对应分类目录
2. 更新文档内容和最后更新日期
3. 更新相关索引文件
4. 在变更日志中记录
